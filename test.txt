Not Yet Integrated / Needs Work

SQLite storage — listed as an option in config (STORAGE_BACKEND=sqlite) but the database.py implementation likely only has in-memory and MongoDB backends; SQLite adapter not written
Redis queue backend — config supports QUEUE_BACKEND=redis but the actual Redis-backed task queue isn't implemented; everything currently runs on the in-memory asyncio queue
LLM-enhanced prompt parsing — LLM_PROVIDER=openai/anthropic is in config but prompt_parser.py likely does regex/keyword extraction only; the LLM path isn't wired
YellowPages crawler — SourceType.yellowpages exists in the model and scorer gives it a +10 bonus, but there's no actual YellowPagesCrawler — only the classifier recognises the domain
Contact enricher hasn't been live-tested — the code was written and wired this session but not validated end-to-end against real JustDial listing pages; phone/email extraction success rate is unknown
Authentication — the API has no auth layer; anyone with the URL can trigger jobs and read all leads
Webhook / job completion callback — no way to get notified when a job finishes without polling
Email/phone validation libraries — phonenumbers and email-validator are in requirements.txt but the normalizer doesn't use them for proper E.164 formatting or MX record validation
Frontend quality — the UI exists but hasn't been tested for edge cases (empty results, long-running jobs, error states)
Result caching — identical prompts re-run the full pipeline every time; no cache layer to skip redundant crawls
